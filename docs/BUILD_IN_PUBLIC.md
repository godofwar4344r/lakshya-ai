## POST 3 · Friday — The "what I learned" post

### LinkedIn version

```
5 hackathon lessons from building Lakshya AI solo this week:

1. Cut scope ruthlessly. I started with 5 product surfaces. Cut to 3. Then collapsed those 3 into ONE workspace with role-switching. Final result is sharper than the original plan.

2. OpenAI streaming + JSON mode is the sweet spot. GPT-4o streaming for teacher tools (real-time = magical UX). GPT-4o-mini with JSON mode for student feedback (fast + structured rubrics in <2s).

3. Pre-cache your demo. I built 3 ready-to-show Physics templates with cached responses. If the API hiccups during my pitch, the demo still works. Belt + suspenders.

4. Mock SSO is product theater that costs nothing. A 1.8-second "Authenticating with Delhi Public School..." overlay signals enterprise-readiness to judges and buyers.

5. The B2B narrative beats the consumer one. I started pitching "teachers + students learn AI". Switched to "institutions measure their AI readiness". The second framing is what admins actually buy.

Phase 1 demo tomorrow. ₹100cr ambition. Day 1.

#BuildInPublic #OpenAI #AIInEducation
```
